# v1.3 Phase A — 教育 + 饱腹度 slider + 成就 skeleton + 体重对比

**目标**:把 mealmate 从「打卡工具」升级到「让用户理解自己吃了多少 / 走了多远」的
**教育型陪伴 app**,同时打下成就系统 skeleton 让下一 phase 能快速 ship 更多成就。

不改核心 pure encouragement 原则(见 [`v1.2.5-pure-encouragement.md`](../v1.2.5-pure-encouragement.md))。
新加东西必须遵守:无扣分、无沮丧 mascot、可自由跳过。

---

## 1. 饱腹度 0-10 slider(替换 3 档 picker)

### 现状

- `app/src/components/ui/FullnessRatingPicker.tsx` — 3 选 1 卡片(3/5/8 分)
- `types/index.ts`: `FullnessScore = 3 | 5 | 8`
- `store/useStore.ts` addFullnessRecord({slot, score: FullnessScore})
- `fullnessHistory: FullnessRecord[]` 存 3/5/8 三档
- 触发点:餐后一段时间(现在没主动 flow,靠 dev / records tab 手动;待 clarify)

### 新设计

**UI:横向 slider 0-10,3 个大 label 0 / 5 / 10 + 实时数字浮字**

```
┌──────────────────────────────────────────┐
│  你现在多饱?                              │
│                                          │
│         [●]  6                           │
│  ○────────●───────────────────○          │
│  0        6                  10          │
│                                          │
│  「记录你有多饱,可以帮你理解每天真正吃了多少」│
│                                          │
│  [ 记下来 ]                              │
└──────────────────────────────────────────┘
```

- 拖动实时更新数字气泡(reanimated worklet)
- 松手 → 数字定格 + mascot 弹一句反馈:
  * 0-2:「今天吃得比较少呢,是不是没胃口?」+ 关心 tone
  * 3-5:「吃了半饱,后面可以再补一点」+ 中性 tone
  * 6-8:「吃得挺好,身体记住这个感觉」+ 鼓励 tone
  * 9-10:「今天挺满足的!」+ 庆祝 tone(用 celebration mascot)

**教育文案永久放在 slider 下方**(不是一次性),让用户每次看到都被暗示"这不是评分,是自我观察"。

**通关规则 chip 加一条**:
- Stage 2:「饱腹度 6-8 的餐算优质餐,不强求每次都到 8+」
- Stage 3+:「持续记录饱腹度可以帮你识别身体信号」

## 2. Onboarding「你现在吃几顿?」步骤

### 现状

Onboarding 4 步:eating → profile → schedule → name(最后落 finishOnboarding)。
schedule 步固定问 3 餐时间(早/午/晚)。

### 新设计

**新加第 5 步**:插在 profile 和 schedule 之间。

```
┌───────────────────────────────┐
│  你现在一天吃几顿?             │
│                               │
│  ┌───────┐   ┌───────┐        │
│  │  2 顿  │   │  3 顿  │        │
│  └───────┘   └───────┘        │
│                               │
│  「没关系,我们从你的节奏开始」  │
│                               │
│  [ 下一步 ]                    │
└───────────────────────────────┘
```

- 2 顿:早+晚 / 午+晚(下步 schedule 让 user 选哪 2 顿)
- 3 顿:早+午+晚(默认)

Schedule 步根据 `dailyMealCount` 显示对应 slot 时间选择。missedScan 也读该字段
决定哪些 slot 算 pending(比如 2 顿用户跳过午餐不算 missed)。

### Store 改动

```ts
type State = {
  // ...
  dailyMealCount: 2 | 3;
  enabledSlots: MealSlot[]; // 2 顿时用户选的具体 slot,3 顿时 = ['breakfast', 'lunch', 'dinner']
};

setDailyMealCount: (n: 2 | 3) => void;
setEnabledSlots: (slots: MealSlot[]) => void;
```

Migrate v16→v17:老用户默认 `dailyMealCount = 3`,`enabledSlots = ['breakfast', 'lunch', 'dinner']`。

### 文件清单

| 文件 | 改动 |
|---|---|
| `src/store/useStore.ts` | 加 2 字段 + setter + migrate |
| `app/onboarding/meal-count.tsx` **新** | 新一步 UI |
| `app/onboarding/_layout.tsx` | 加 route 到 stack |
| `app/onboarding/profile.tsx` | onNext push meal-count |
| `app/onboarding/schedule.tsx` | 按 enabledSlots 只显示对应 slot |
| `src/services/missedScan.ts` | 过滤只扫 enabledSlots 里的 slot |
| `src/store/selectors/reminder.ts` | 同上 |
| `app/(main)/settings.tsx` | 加「一天几顿」section 让用户随时改 |

**预计 3 commits**:
1. store schema + migrate + onboarding UI + route
2. 后端 selector / missedScan 读 enabledSlots
3. settings section 让老用户能改

### Sim 测试 case

- [ ] 新装:onboarding profile → meal-count → 2 → schedule 只显示早+晚 2 slot
- [ ] 新装:meal-count → 3 → schedule 显示早/午/晚 3 slot
- [ ] 老用户 v16→v17 migrate:home 仍显示 3 slot(dailyMealCount=3 default)
- [ ] Settings 里切 3→2 后 home 立刻反映

### 开放决策

- **OPEN-A3**:2 顿时具体哪 2 顿?让用户任选组合 or 只给 2 组默认(早晚 / 午晚)?
  - **推荐**:让用户任选任意 2 slot(灵活),默认预选早+晚

## 4. 初始 vs 当前体重对比卡(Stage 2/3 home)

### 目标

让用户直观看到「从起点到现在走了多远」,强化「一直在向健康方向」的正反馈。

### Selector

```ts
export function selectInitialVsCurrentWeight(state: State): {
  initial: { kg: number; date: string } | null;
  current: { kg: number; date: string } | null;
  delta: number; // current - initial(可负)
  daysSinceInitial: number;
} | null {
  const sorted = [...state.weightHistory].sort((a, b) => a.date.localeCompare(b.date));
  if (sorted.length < 1) return null;
  const initial = sorted[0];
  const current = sorted[sorted.length - 1];
  // ...
}
```

### UI

放在 Stage 2 / Stage 3 home 里,「当前体重」卡下方(如果只有 1 条记录不显示)。

```
┌──────────────────────────────────────┐
│ 从起点到现在                          │
│                                      │
│   起点: 65.0kg  →  现在: 67.5kg      │
│   06-01              06-21           │
│                                      │
│   增了 2.5 kg,3 周,每天在慢慢向健康  │
│   走近 🌱                            │
└──────────────────────────────────────┘
```

Mascot 文案分 3 种(不出现「减了 / 少了 / 掉了」类字):
- delta > 0(增重):「增了 X kg,X 天,每天在慢慢向健康走近 🌱」
- delta == 0:「保持得挺好,X 天里稳稳的」
- delta < 0(减重):「变化 X kg,X 天,身体在自己调节 🌿」(去 evaluate)

### 文件清单

| 文件 | 改动 |
|---|---|
| `src/store/selectors/weight.ts` **新** | selectInitialVsCurrentWeight |
| `src/components/home/WeightCompareCard.tsx` **新** | 卡片 UI |
| `src/components/home/HomeStage2.tsx` | 集成 |
| `src/components/home/HomeStage3.tsx` | 集成 |

**预计 2 commits**:
1. selector + card 组件
2. 集成 Stage 2/3

### Sim 测试 case

- [ ] 只有 1 条体重 → card 不显示(避免 delta = 0 误导)
- [ ] 2 条 → card 显示 initial + current + delta
- [ ] delta > 0 / == 0 / < 0 各拿到对应文案
- [ ] Stage 4/5 home 不显示这个 card(它们自己有别的体重展示)

### 开放决策

- **OPEN-A8**:Stage 5 用户是不是也想看这个 card?他们已经在目标 ±2.5kg,
  「起点到现在」可能就是 stage-1 的减重旅程。
  - **推荐**:Stage 5 也显示,但文案换成「你已经走了这么远 🌿」类庆祝调
- **OPEN-A9**:delta < 0(减重)对 stage 3「健康增重」用户 = 坏消息。是否要
  换成担心 tone?
  - **推荐**:**不换**,还是中性「身体在自己调节」。担心 tone = 惩罚感。
    加一个安慰次要句「如果最近有变化,记得跟自己聊聊」
