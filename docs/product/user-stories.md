# User Stories

从 PRD §1-§4 五阶段定义反推。按角色 + 行为 + 价值组织。

## 核心角色

- **饮食不规律者**：经常跳过正餐、忙起来忘了吃、一天只吃一顿
- **恢复期 / 食欲差用户**：身体恢复中、需要重新建立三餐节奏
- **健康增重者**：偏瘦、希望长肉但不知道怎么吃
- ⚠️ **敏感人群**：与进食障碍康复人群有重叠（PRD §八 强调谨慎设计）

## Stage 1 — 坚持

1. **作为**饮食不规律的用户，**我希望**有人在三餐时间提醒我，**这样**就不会一忙起来又忘了吃。
2. **作为**新用户，**我希望**有一个温柔的伙伴陪我吃饭而不是冷冰冰的打卡 App，**这样**坚持的心理门槛能降低。
3. **作为**用户，**我希望**拍照打卡能立刻看到正向反馈（HP +5 + 鼓励台词），**这样**知道我在前进。
4. **作为**偶尔会错过的用户，**我希望**错过餐扣分不那么重 + 文案温柔，**这样**不会因为一次失败就放弃。
5. **作为**好奇下一步是什么的用户，**我希望**有一个明确的阶段升级目标（HP 满 100 解锁 Stage 2），**这样**有持续动力。

## Stage 2 — 量化

6. **作为**进阶用户，**我希望**记录每餐的饱腹度（不只是"吃了"），**这样**伙伴能更了解我状态。
7. **作为**关心体重变化的用户，**我希望**每天拍体重秤照片自动归档，**这样**不用手动记 Excel。
8. **作为**希望看长期趋势的用户，**我希望**统计页能看到 HP / 体重曲线，**这样**知道走了多远。
9. **作为**偶尔来不及拍体重秤的用户，**我希望**有开关跳过体重秤照片，**这样**不影响录入。

## Stage 3 — 健康增重

10. **作为**偏瘦希望增重的用户，**我希望**每周净增 1kg 能被加分认可，**这样**有方向感。
11. **作为**接受运动建议的用户，**我希望**每周 3 次 × 3 小时的目标既不激进也能形成习惯，**这样**不会半路放弃。

## Stage 4 — 营养（TBD）

12. **作为**已经能稳定吃饱的用户，**我希望**关注吃的结构（主食 / 蛋白 / 蔬果），**这样**质量也跟上。
13. **作为**怕麻烦的用户，**我希望**不必精确到克数、用轻量标签就能记，**这样**愿意每天用。

## Stage 5 — 持之以恒（TBD）

14. **作为**长期用户，**我希望**没有新的硬约束，**这样**习惯能过成日子。
15. **作为**断了几天又回来的用户，**我希望**伙伴还是温柔等我而不是责备，**这样**不会有"我已经放弃了"感。

## 跨阶段 / 设置

16. **作为**生活节奏特殊的用户（比如晚睡 / 倒班），**我希望**可以自定义三餐推送时间，**这样**适配我的真实作息。
17. **作为**情绪敏感的用户，**我希望**能开启温柔模式（扣分减半 / 文案更柔），**这样**用起来不焦虑。
18. **作为**注重隐私的用户，**我希望**所有数据本地存（v0.4 当前），未来上云时也明确告诉我什么数据上了云，**这样**有控制感。
19. **作为**想清空重来的用户，**我希望**有"重置数据 / 删除账号"按钮，**这样**心理压力小。
