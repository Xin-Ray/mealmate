## 改动 2 — NextMealCard 时间不实时 + 不准确

### 根因
- NextMealCard 内 `[, setTick] = useState(0)` ignored value,React Compiler
  可能当死代码消掉 → useEffect setInterval 不重建 → 倒计时冻结
- HomeMealStatusSlot 完全没 ticker → 在 NextMealCard / MealReminderCard /
  MealIncompleteCard 之间的切换决策只在 mount 时算一次 → 用户开 home 不动,
  meal schedule 跨越时分支不切

### Fix
- NextMealCard:`useState(() => Date.now())` 真 state + selector 显式接 now
- HomeMealStatusSlot:30s setInterval tick + `void nowMs` 防 RC dead-code

### 测试 case [真机:必跑]

## 改动 4 — Launch 动画 — 等 xin 拍板

### 视频信息
- 文件: `30f98e63-video_1.mov`(xin 上传)
- 时长: **8.04 秒**(`afinfo`)
- 大小: 6.4 MB

### 实现方案 trade-off

| 方案 | pros | cons | 推荐场景 |
|---|---|---|---|
| **A. expo-av Video 播 mp4** | 简单,质量好 | 需新依赖 `expo-av`(~3MB);8 秒每次播太长 | 只第一次启动播,后续静态 |
| **B. Lottie JSON** | 文件小,可控 | 需 AE 资源 xin 没有 | 后续考虑 |
| **C. 帧序列 PNG + expo-image-manipulator** | 无新依赖 | 代码量大,文件可能更大 | 不推荐 |
| **D. WebP 动图 / GIF** | 简单 | 解码慢,质量差 | 不推荐 |

### 推荐路径(等 xin 拍板)

**A + AsyncStorage flag**:
- 装 `expo-av`
- `ffmpeg -i video_1.mov -c:v libx264 -c:a aac launch.mp4`(~2-3 MB 后)
- 放 `app/assets/animations/launch.mp4`
- `app/_layout.tsx` 加 `<LaunchAnimation />`,首次启动 AsyncStorage 没 `hasSeenLaunch` flag → 全屏覆盖播放 → 视频结束 setItem + dismiss
- 老用户(已 onboarding 完)也算"已见",不重复播
- 8 秒太长 → **可截前 3 秒**(`ffmpeg -t 3`)做更短 splash

### Open 问题(xin 拍)
- 是否每次冷启动都播(8 秒太长,推荐截短)还是只首次
- 是否允许新依赖 expo-av(轻微包大小)
- 是否能/想截短到 ~3s
