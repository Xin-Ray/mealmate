## 改动 2 — NextMealCard 时间不实时 + 不准确

### 根因
- NextMealCard 内 `[, setTick] = useState(0)` ignored value,React Compiler
  可能当死代码消掉 → useEffect setInterval 不重建 → 倒计时冻结
- HomeMealStatusSlot 完全没 ticker → 在 NextMealCard / MealReminderCard /
  MealIncompleteCard 之间的切换决策只在 mount 时算一次 → 用户开 home 不动,
  meal schedule 跨越时分支不切

### Fix
- NextMealCard:`useState(() => Date.now())` 真 state + selector 显式接 now
- HomeMealStatusSlot:30s setInterval tick + `void nowMs` 防 RC dead-code

## 改动 4 — Launch 动画 ✅ 已实现(A.2 默认)

### 决策
- xin「继续」语境下默认推 A.2:**每次冷启动都播 ~3s 标志性 splash**
- 视频源 `video_1.mov`(8.04s, 6.4MB)用 `avconvert` 截前 3s + Preset960x540
  → `app/assets/animations/launch.mp4`(1.1 MB)

### 实现
- `app/assets/animations/launch.mp4` — 3s 视频(avconvert 转,无 ffmpeg)
- 装 `expo-av@~16.0.8`(对齐 SDK 54)
- 新 `app/src/components/launch/LaunchAnimation.tsx`:
  - `<Video />` 全屏覆盖
  - `shouldPlay isLooping={false}` 播完触发 `didJustFinish` → onFinish
  - Pressable 包裹 → 点屏跳过(不耐烦的用户也能立刻进 app)
- `app/app/_layout.tsx`:
  - `const [launchAnimDone, setLaunchAnimDone] = useState(false)`
  - 渲染最上层 `{!launchAnimDone && <LaunchAnimation onFinish={() => setLaunchAnimDone(true)} />}`
  - 覆盖在 Stack 之上,不影响 onboarding / home / modal 导航(unmount 后让出)

### 注意
- expo-av 是 native 模块,需要 native rebuild。
  - dev sim 测试:`npx expo run:ios` 重 build dev client(否则 Video 不渲染)
  - EAS Build production:自动包含新 native 模块,直接打包
- 视频每次冷启动播 3s,**冷启动总时长增加 ~3s**(用户感知:更有 brand 感)
- 后台再回前台 ≠ 冷启动,不会重播(React state 在内存,launchAnimDone 持久)

### 测试 case
- [ ] [真机 EAS Build] 冷启动 → 全屏视频 3s → **400ms fade out** → onboarding/home
- [ ] [真机] 视频播到一半 → 点屏 → **400ms fade out**(不是硬切)→ dismiss
- [ ] [真机] 进 app 后切后台 → 再回前台 → **不重播**视频
- [ ] [真机] kill app + 重启 → 重新播 3s + fade
- [ ] [sim 重 build dev client] 同上 4 case
- [ ] [真机] 连续点屏 2 次 → 只触发一次 fade(fadeStartedRef 防抖)

### Trade-off 记录
- 包大小增加 ~1.1 MB(launch.mp4) + ~3 MB(expo-av native + JS) ≈ 4 MB
- 不依赖 ffmpeg(用 macOS 内置 avconvert)
- 视频带音频(`isMuted={false}`)— 若 xin 觉得吵改 `isMuted={true}` 即可
