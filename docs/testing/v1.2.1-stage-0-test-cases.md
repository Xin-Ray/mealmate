## React Compiler + Pressable 已知坑

**症状**:Pressable 的 `style={({pressed}) => ({...})}` 形式在 React Compiler
启用下被 swallow,导致按钮 bg / border / 所有 style props 丢失,只渲染 `<Text>`
子节点。视觉上看起来「按钮消失了」。

**复现**:`app.config.ts` 里 `experiments.reactCompiler: true` + Pressable 用
函数式 style。Sim 真机都中招。

**Workaround**:用 plain object style:`style={{...}}`(放弃 pressed 视觉反馈)。
或用 className(NativeWind)代替。

**已知影响范围**(全部已修):
- `app/src/components/home/HomeStage0.tsx` CTA — v1.2.2 P0 fix
- `app/src/components/ui/SnackCard.tsx` press → bg 切色 — v1.2.2 fix
- `app/src/components/stage/SimpleTransitionScreen.tsx` 继续按钮 — v1.2.2 fix

未来加新 Pressable **务必避免**这种函数式 style 形式。
