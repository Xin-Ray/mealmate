## F3 · Stage 4「距目标 X%」公式错

| 字段 | 值 |
|---|---|
| 类别 | 🔴 HIGH 数据错 |
| 来源 | xin 装机看 stage 4 hero 圆环 + "距目标 Xkg" 提示，X% 数字错 |
| 当前实现 | `WeightGoalProgressCard`: `pct = current / target`，"距目标" 显示 `(target - current).toFixed(1)` |
| 应改为 | xin 给的公式 `(target - current) / (target - initial) * 100`<br/>例子：start=60, target=65, current=62 → `(65-62)/(65-60) = 3/5 = 60%`<br/>**语义 = 「还差 X%」**（current 越接近 target，% 越小） |
| 🚩 OPEN-R1-A | UI label 当前是「距目标 Xkg」+ 圆环里显示「N%」。新公式下 N 代表「还差%」。**圆环应该填「完成%」还是「剩余%」？** 我的默认：圆环填 `1 - 还差% = 完成%`（更符合"进度条向满"直觉），label 显示「还差 N%」+ 「距目标 Y kg」 |
| 🚩 OPEN-R1-B | `initialWeight` 哪里来？候选 (a) `weightHistory[0].kg`（第一条记录）；(b) 用户设 targetWeight 时刻的体重快照（需新字段 `initialWeightAt` + `initialWeightValue`）。**默认选 (a)** —— 简单，不动 schema |
| 边界 | initial 缺失（无 weightHistory）→ 公式回 `null`，显示 "—"；current 已 ≥ target → pct=100%，"还差 0%" |
| 涉及文件 | `app/src/components/home/stage4/WeightGoalProgressCard.tsx` |
| 预期 commit | `fix(stage-4): F3 圆环进度公式 = remaining gap / total gap` |
