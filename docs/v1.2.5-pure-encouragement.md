# v1.2.5 — 全 stage 取消扣分 (pure encouragement)

**xin 原话**:「不然很伤心的」。

跟 PRD §八 安全与伦理边界 §1 一脉相承(「永远不把 HP 下降描述为『你失败了』」),
v1.2.5 把这条原则推到底:任何 stage 的任何操作都**只奖励,不惩罚**。

## 设计 shift

| 机制 | v1.2.4(老) | v1.2.5(新) |
|---|---|---|
| Stage 1/2/3 漏餐 | `stageScore -10`(clamp 0) | **不动** |
| Stage 4 漏餐 | `HP -10`,HP<0 → demote 回 stage 3 | **HP 不动** |
| Stage 5 某日超体重 | `stars -1`,stars=0 → demote 回 stage 4 | **stars 不动** |
| Stage 5 永久 demote 路径 | 存在 | **无**(stars 只升不降) |
| 漏餐 dialogue 文案 | 「你错过了一餐,有点饿」「贵在坚持」(略带责备/压力) | 「这餐没赶上,下一顿见 🌱」「慢慢来,不急」(中性鼓励) |
| 漏餐 dialogue `hpDelta` | -10 / -5 (gentle) | **0**(UI 不渲染血量 badge) |
| Meal record `hpDelta`(missed) | -10 / -5 | **0** |
| Stage 0/0.5 漏餐 | 已免疫(v1.2.1 起) | 不变,继续免疫 |
| `demoteStage` action | 仍存在,由 addHp(<0)/stage5 触发 | 保留函数,但**无 caller**(dead 但保留备用) |
| `gentleMode` 字段 | 控制漏餐扣分折半 | 保留(字段+ setter),但漏餐路径已不读 |
| WeekStrip missed cell | 红/灰显示 | 灰显示(无变化) |
| 通关条件(stage 1/2/3) | `stageScore >= target` | **不变**(40/50/30) |
| 通关条件(stage 4) | 体重 ≥ standardWeight | **不变** |
| 通关条件(stage 5) | 60 天 + 仍未 demote | **改成纯 60 天**(无 demote 路径) |

## 没改的地方

- `demoteStage` 函数本体 — 保留备用(如未来要 reintroduce 加 feature flag 切)。
  实际不会触发:`addHp` 现在只收正 delta(markMealDone +10、addSnack +10),
  `__internal_runStage5Check` 也不再调它。
- `gentleMode` 字段、setter、Settings 开关 — 字段保留,UI 保留。设置仍可切,
  但效果消失(漏餐路径已删 gentleMode 分支)。后续要么删这个 UI、要么给它新含义。
- `RecordCard` / `TodayRecordRow` 血量 badge 渲染 — 守卫已是 `hpDelta < 0`,
  现在 `hpDelta = 0` 自然不渲染,无需改 UI。
- `dialogue.kind === 'failure'` / `pushDialogue(failure)` — 仍存在但不会触发
  (demoteStage 不再被调)。feed render 走暖橘卡的分支保留。

## 风险 / 留账

- **作弊面**: 现在用户即使全天不吃,游戏状态也不退。等于「只能升、不能降」。
  接受 — xin 的 product call: 这是 wellness 工具,不是积分赛。
- **gentleMode 失效**: UI 仍可切但已无效。下一版应该删 Settings 该开关 +
  字段 + setter。本期不删避免污染 hotfix scope。
- **demoteStage 死代码**: 同上,保留备用。
- **failure dialogue kind**: 同上。
- **dev menu `__dev_setHp(0)`**: 在 stage 1/2/3 这个值现在不进通关路径
  (改 stageScore 才是),设了不会触发 demote 因为 addHp(0) 不会让 unclamped<0。
  在 stage 4/5 dev 直接设 hp=0 仍会触发 demote(addHp 路径)? 不会,
  `__dev_setHp` 是 `set({ hp: clampHp(n) })` 直接落,不走 addHp,所以不触发。安全。
