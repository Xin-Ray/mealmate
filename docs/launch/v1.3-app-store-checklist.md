## 1. 法务文档(必须,提交审前 host 在公网)

### 1.1 隐私政策 `privacy.html`

**必须**能被 App Store Connect 里填的 URL 访问到。可以 host 在 GitHub Pages /
一个静态站(mealmate.xin/privacy 之类)。中英双语。

**必包含 12 项(Apple 要求)**:

- [ ] app 名称 + 开发者名 + 联系邮箱
- [ ] 收集哪些数据(枚举):
  * 用户输入(体重、餐时、饱腹度、身高、性别、目标体重)
  * 相机 / 相册权限 → 只本地存储照片路径,不上传到 server
  * Apple ID 邮箱(仅 Sign in with Apple 用)
  * 云端同步存储:餐记录 / 体重 / 对话历史(如果启用了云同步)
  * **不收集**:定位、通讯录、健康 App 数据、广告 ID、任何设备指纹
- [ ] 用途(each purpose):
  * 记录 / 展示用户饮食进度
  * 跨设备同步(仅登录用户)
  * 客服排障(用户主动联系时)
- [ ] 数据保留期(多久删除)
- [ ] 第三方共享:**无**(如果有 Sentry / Amplitude 之类 SDK 要列)
- [ ] 用户权利:
  * 查看自己所有数据
  * 请求删除账号(会连 backend 一起清)
  * 撤回同意(退出登录 / 卸载 app 后本地数据仍在设备,云端数据可请求删)
- [ ] 儿童隐私(如果 age rating <13):**app 不面向 13 岁以下**,如果检测到未成
      年直接拒绝
- [ ] 跨境传输(如 backend 在美国 / 中国大陆):明确 host 位置
- [ ] Cookie / 追踪:无
- [ ] 变更通知机制
- [ ] 生效日期

## 3. 收费模式(3 候选)

### 候选 A:完全免费 + 未来广告(v1.3 不做)

**Pros**:门槛最低,冷启动最快
**Cons**:未来广告干扰用户体验,跟「陪伴」调性冲突

### 候选 B:免费 core + 订阅高级功能

**Pros**:符合业内常见模式,收入可预期
**Cons**:「订阅陪伴」听起来功利,可能伤 tone。要设计好哪些是免费

**如果走 B,免费 vs 付费候选划分**:
- 免费:Stage 0-3 全功能 / 3 餐记录 / 基础 stats / 通关规则
- 付费(¥9/月 or ¥68/年):Stage 4-5 / 云同步 / 成就 / 高级 stats(月/年报告) / 饱腹度 slider

### 候选 C:一次性买断(¥18-38)

**Pros**:tone 干净,买了永久用,不搞订阅心智
**Cons**:收入天花板低,后续需要不断出付费大版本
**推荐 C** — 跟 mealmate 陪伴调性最贴,一次买断像收养一个伙伴
