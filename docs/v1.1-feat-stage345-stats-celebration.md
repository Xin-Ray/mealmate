### Schema migrate v10 → v11

```ts
if (version < 11 && ps.targetWeight === undefined) {
  ps.targetWeight = null;  // 老用户回填 null（stage 1/2/3 用不到）
}
```

### 进阶 / 降阶触发器

**位置**：`addWeightRecord` action 末尾跑判定（**只在记录体重时跑**，不在 missedScan / markMealDone 等其它路径跑）。

```ts
addWeightRecord: ({ kg, photoUri }) => {
  // ...existing logic: append to weightHistory + addHp(+0.5)...

  const s = get();
  const target = s.targetWeight;
  if (target == null) return;  // 没设目标 → 不触发任何 stage transition

  // Stage 4 → 5: 当前体重 ≥ target
  if (s.currentStage === 4 && kg >= target) {
    get().advanceStage();   // 现有 action，已 push pending stage-4-end
    return;
  }

  // Stage 5 完成：30 天内 weight 全部 ∈ [target-2.5, target+2.5]
  if (s.currentStage === 5) {
    const lookback = 30 * 24 * 60 * 60 * 1000;
    const cutoff = Date.now() - lookback;
    const recent = s.weightHistory.filter(w => w.recordedAt >= cutoff);
    if (recent.length >= 30) {  // 至少有 30 条记录
      const allInRange = recent.every(w =>
        w.kg >= target - 2.5 && w.kg <= target + 2.5
      );
      if (allInRange) {
        // TODO-4：stage 5 完成态怎么标？暂时 push 一个特殊 transition
        get().__internal_completeStage5();
      }
    }
  }
};
```

**注意**：
- Stage 3 → 4 进阶仍走原有 HP=100 路径（PRD §四 没改）
- 不动 `advanceStage` / `demoteStage` 内部逻辑
- Stage 5 完成后体重再超 → demote 回 stage 4（占位实现 TODO-5）

## 八、拍照庆祝动画

### Figma 参考：`32:1637`

白色圆角弹窗 on photo bg：
- 顶部 mascot 立绘
- 两侧 ❤️ 心形装饰（**不是** confetti，符合敏感人群约束）
- 文案：`"太棒了！早餐看起来很健康呢！"`（按 slot 切换）
- HP delta：`💚 血量 +5`
- 按钮：`"继续加油！"`

### 动画实现（Reanimated v4，无新库）

**时序**（总 600ms）：

```
t=0     :  弹窗出现（scale 0.85 → 1.0，opacity 0 → 1，180ms ease-out）
t=120ms :  mascot 弹跳（translateY 0 → -8 → 0，2 次循环，total 240ms cubic-bezier）
t=200ms :  "+5" 浮字（translateY 0 → -20，opacity 1 → 0，400ms ease-in）
t=200ms :  心形左右轻摆（rotate ±10°，3 次，total 400ms）
t=600ms :  动画静止；button "继续加油" enabled
```

**用户交互**：
- 任意时刻点屏 → 立即跳到末态（终止所有动画）
- 点"继续加油" → 弹窗 fade out 150ms → 回 home

**变体**：
- 餐打卡：`💚 血量 +5`
- 加餐：`💚 血量 +10`
- 文案：按 mealSlot 切（早 / 午 / 晚 / 加餐）+ HP band（满血 / 平稳 / 低 / 濒临）

**敏感人群约束**：
- 不用 confetti
- 不用"赢了/成功/失败/你让我失望"
- mascot 表情温和（不夸张大笑）
- 文案"太棒了"已经在边界，TODO 等 xin 复核语气

### 集成位置

`app/app/(modal)/photo.tsx` result phase 改造：
- 现有 result UI 整体替换为 `<CelebrationModal>`
- 加餐分支（snack=true）同样走 CelebrationModal，HP delta=+10
