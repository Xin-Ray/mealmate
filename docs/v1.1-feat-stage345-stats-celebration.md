## 二、xin 拍板的全部决策（2026-05-29）

### Stage 3 — 健康增重 v1

沿用 stage 2 骨架（吃饱三餐 + 体重 + 加餐），hero 文案 + mascot 反映「健康增重 v1」语境。

### Stage 4 — 营养（进阶条件）

**当前体重 ≥ standardWeight → 自动进阶 stage 5**

`standardWeight` 不是用户自设的 `targetWeight`，是按身体数据反推的"理想体重"：

```
standardWeight = bmi × height_m²

其中 bmi:
  asian       → 21    (中国 / 东亚优化值)
  其它 ethnicity → 22  (国际通用)
```

**Gender**：字段先加但**不参与公式**（v1.1）。后续可改 Devine 公式（男 +2kg / 女 -1kg），列为 future TODO。

### Stage 5 — 持之以恒（精确化版）

进入 stage 5 后开始累计 **stars**：

| 触发 | 操作 |
|---|---|
| 每 7 天体重全部 ∈ `[targetWeight - 2.5kg, targetWeight + 2.5kg]` | **+1 星** |
| ~~某日体重 > `targetWeight + 2.5kg`（超体重）~~ | ~~**-1 星**~~ — **v1.2.5 起取消** |
| ~~星数 = 0~~ | ~~**demote 回 stage 4**~~ — **v1.2.5 起取消**(stars 只升不降) |
| 持续 60 天 | **stage 5 完成 → push stage-5-end** |

> v1.2.5 design shift: stars 只升不降。Stage 5 永不 demote 回 stage 4。详见
> [v1.2.5-pure-encouragement.md](./v1.2.5-pure-encouragement.md)。

**对比** 之前 doc 里的占位「30 天体重全在区间」—— xin 给的实际方案更精细，引入了星数累计 + 周期判定。

## 五、Store 改动

### 新增字段（schema v10 → v11）

```ts
type Gender = 'male' | 'female' | 'other';
type Ethnicity = 'asian' | 'other';

type StageHistoryEntry = {
  ts: number;          // ms timestamp
  stage: 1 | 2 | 3 | 4 | 5;
  reason: 'advance' | 'demote' | 'init';
};

type State = {
  // ...existing...

  // ★ 用户体质数据（v11） — standardWeight 公式输入
  height: number | null;          // cm
  gender: Gender | null;          // 加字段但 v1.1 不参与公式（future Devine）
  ethnicity: Ethnicity | null;    // asian → BMI 21；其它 → BMI 22

  // ★ 用户目标 — stage 5 区间判定 ±2.5kg 用
  targetWeight: number | null;    // kg

  // ★ Stage 历史 — stats tab "等级变化图表" 用
  stageHistory: StageHistoryEntry[];

  // ★ Stage 5 状态机
  stage5StartedAt: number | null; // 进入 stage 5 时刻（advance 后 set）
  stage5Stars: number;            // 当前星数；初始 0，60 天累计；0 → demote
  stage5LastStarCheck: number | null; // 上次"每 7 天"判定时刻
};
```

### Schema migrate v10 → v11

```ts
if (version < 11) {
  ps.height ??= null;
  ps.gender ??= null;
  ps.ethnicity ??= null;
  ps.targetWeight ??= null;
  // 老用户 stageHistory 回填：根据 currentStage 假装从 init 一路 advance
  if (!Array.isArray(ps.stageHistory)) {
    const stage = ps.currentStage ?? 1;
    const now = Date.now();
    ps.stageHistory = [];
    ps.stageHistory.push({ ts: now, stage: 1, reason: 'init' });
    for (let s = 2; s <= stage; s++) {
      ps.stageHistory.push({ ts: now, stage: s, reason: 'advance' });
    }
  }
  ps.stage5StartedAt ??= null;
  ps.stage5Stars ??= 0;
  ps.stage5LastStarCheck ??= null;
}
```

### 派生 selector

```ts
// app/src/store/selectors/standardWeight.ts
export function selectStandardWeight(state: {
  height: number | null;
  ethnicity: Ethnicity | null;
}): number | null {
  if (state.height == null) return null;
  const heightM = state.height / 100;
  const bmi = state.ethnicity === 'asian' ? 21 : 22;
  return Math.round(bmi * heightM * heightM * 10) / 10;  // 1 位小数
}
```

### 改动 `advanceStage` / `demoteStage`

**不动内部逻辑**，只在末尾 append `stageHistory`：

```ts
advanceStage: () => {
  // ...existing: stage += 1, hp = HP_INITIAL_STAGE_X, push pending end transition...
  const newStage = get().currentStage;
  set(s => ({
    stageHistory: [...s.stageHistory, {
      ts: Date.now(), stage: newStage, reason: 'advance'
    }]
  }));
  // 副效果：如果新 stage = 5 → 初始化 stage5 状态
  if (newStage === 5) {
    set({ stage5StartedAt: Date.now(), stage5Stars: 0, stage5LastStarCheck: null });
  }
},

demoteStage: () => {
  // ...existing...
  const newStage = get().currentStage;
  set(s => ({
    stageHistory: [...s.stageHistory, {
      ts: Date.now(), stage: newStage, reason: 'demote'
    }]
  }));
  // 如果是从 5 → 4，重置 stage5 状态
  if (newStage === 4) {
    set({ stage5StartedAt: null, stage5Stars: 0, stage5LastStarCheck: null });
  }
},
```

### Stage 4 → 5 触发器

**位置**：`addWeightRecord` action 末尾。

```ts
addWeightRecord: ({ kg, photoUri }) => {
  // ...existing logic: append weightHistory + addHp(+0.5)...

  const s = get();

  // Stage 4 → 5: 当前体重 ≥ standardWeight
  if (s.currentStage === 4) {
    const standard = selectStandardWeight(s);
    if (standard != null && kg >= standard) {
      get().advanceStage();  // 已带 stageHistory append + stage5 init 副效果
      return;
    }
  }

  // Stage 5 星数判定（下面单独章节）
  if (s.currentStage === 5) {
    get().__internal_runStage5Check();
  }
},
```

### Stage 5 星数 / demote / end 判定

新 internal action（不暴露给 UI 直接调，由 `addWeightRecord` + 进 home / AppState active 时触发）：

```ts
__internal_runStage5Check: () => {
  const s = get();
  if (s.currentStage !== 5) return;
  if (s.stage5StartedAt == null) return;
  if (s.targetWeight == null) return;  // 没设目标不算

  const target = s.targetWeight;
  const now = Date.now();
  const daysSinceStart = Math.floor((now - s.stage5StartedAt) / (24*3600*1000));

  // 60 天完成
  if (daysSinceStart >= 60) {
    get().advanceStage();  // 跑常规 advance 到 stage 5 + push pending end transition
    // 实际上 advanceStage 内 `if currentStage >= 5 no-op`，所以需要单独 push 一个
    // stage-5-end transition。改动 advanceStage 加 stage 5 完成分支。
    return;
  }

  // 检查今日体重是否超 target+2.5 → -1 星
  const todayWeights = s.weightHistory.filter(w =>
    dateOf(w.recordedAt) === s.todayKey
  );
  const todayMax = todayWeights.length > 0
    ? Math.max(...todayWeights.map(w => w.kg)) : null;
  if (todayMax != null && todayMax > target + 2.5) {
    const newStars = Math.max(0, s.stage5Stars - 1);
    set({ stage5Stars: newStars });
    if (newStars === 0) {
      get().demoteStage();  // 回 stage 4，stage5 状态由 demoteStage 重置
    }
    return;
  }

  // 每 7 天判定一次：过去 7 天体重全在区间 → +1 星
  const lastCheck = s.stage5LastStarCheck ?? s.stage5StartedAt;
  const daysSinceLastCheck = Math.floor((now - lastCheck) / (24*3600*1000));
  if (daysSinceLastCheck >= 7) {
    const sevenDaysAgo = now - 7*24*3600*1000;
    const weekWeights = s.weightHistory.filter(w => w.recordedAt >= sevenDaysAgo);
    if (weekWeights.length >= 7) {  // 至少 7 条记录
      const allInRange = weekWeights.every(w =>
        w.kg >= target - 2.5 && w.kg <= target + 2.5
      );
      if (allInRange) {
        set({ stage5Stars: s.stage5Stars + 1, stage5LastStarCheck: now });
      }
    }
  }
},
```

**注意**：
- Stage 1 → 2 / 2 → 3 / 3 → 4 进阶仍走原有 HP=100 路径（PRD §四 没改这部分）
- 新触发器只接 stage 4 → 5（体重达标）+ stage 5 完成判定（60 天）+ stage 5 demote（0 星）
- 不重写 `advanceStage` / `demoteStage`，只在末尾 hook 副效果

### Stage timeline 渲染（重点）

阶梯线：x 轴是时间，y 轴是 stage 1-5，每次 advance/demote 在对应 ts 处出现阶梯跳变。TrendChart 现版接受连续点，可能需要扩 props 支持 step-after 模式。

**实现风险**：TrendChart 是自绘 SVG（不是 Victory/Recharts 之类），改阶梯线需要看下 path 生成逻辑。**TODO** 实施时评估，必要时新建 `<StepChart>` 专用组件。

### Store 加 hpHistory

```ts
type HpHistoryEntry = { ts: number; hp: number };

// addHp action 内追加
addHp: (delta) => {
  // ...existing clamp + advance/demote 边界判定...
  set(s => ({
    hpHistory: appendCapped(s.hpHistory, { ts: Date.now(), hp: get().hp }, 90 * 5)
    // 上限 90 天 × 平均 5 次/天 = 450 条
  }));
},
```

migrate 给老用户回填 `[]`。
