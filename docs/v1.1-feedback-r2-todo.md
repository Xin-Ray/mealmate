**状态**：当前靠**直接编辑 pbxproj**让 Debug=`.dev` / Release=prod，但 `app/ios/` 在 `.gitignore`，所以：
- 新 clone 仓库后跑 `expo prebuild` 会生成默认 pbxproj（没 `.dev` 配置）
- 已有的本机环境如果不小心跑了 `expo prebuild --clean` 会覆盖手改

**长期 fix**：写 expo config plugin（原型在 `backup/local-env-split-attempt` commit `9d35be2` 的 `app/plugins/withBundleIdSuffix.js`），在 prebuild 时声明式 patch pbxproj。

**责任方**：frontend（plugin 本身是 expo config 范畴，跟 backend 无关）

→ 已写入 README "Bundle ID / 双环境约定" 段最后
